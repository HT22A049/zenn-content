## はじめに

JavaScriptのDOM操作の理解を深めるために、
シンプルなTodoアプリを作成しました。

## 実装した機能

- 入力欄に文字を入れてタスク追加（ボタン + Enterキー）
- タスクごとに削除ボタン
- 関数化して処理を共通化

## コード

### script.js

```js
function addTask () {
	const input = document.getElementById("input-text");
	const inputText = input.value;

	// 入力が空なら何もせずに終了
	if (inputText === "") return;

	// 要素作成
	const li = document.createElement("li");
	const del = document.createElement("button");

	del.textContent = "削除";
	li.textContent = inputText;

	// 削除処理
	del.addEventListener("click", () => {
		li.remove();
	});

	li.appendChild(del);
	document.getElementById("item-list").appendChild(li);

	// 入力リセット
	input.value ="";
}

// クリックで追加
document.getElementById("add-btn").addEventListener("click", addTask);

// Enterでも追加
const input = document.getElementById("input-text");
input.addEventListener("keydown", (e) => {
	if (e.key === "Enter") {
		addTask();
	}
});
```

## 解説

### タスク追加

HTML にあらかじめ書くのではなく、
ユーザーの入力に応じて要素を追加する必要があるため、
createElement を使用して動的に作成しています。

### 削除機能

remove() を使用することで、
対象の要素 (li) を DOM から削除しています。

### Enterキー対応

クリックイベントとは別に、
キーボード操作にも対応するために keydown イベントを使用しています。

e.key === "Enter" で Enter キーが押されたことを判定しています。

## 学んだこと

今回の開発では以下のことを学びました。

- addEventListener を使用したイベント処理の書き方
- createElement / appendChild による DOM 操作
- クリックと Enter キーでイベントの種類が異なること

特に、同じ処理を addTask 関数にまとめることで、
コードの見通しが良くなると感じました。

最初は同じ処理なのにクリックとEnterキーで別々にイベントを書く必要があることに疑問を持ちました。

しかし、クリックはマウスイベント、Enterキーはキーボードイベントと、
イベントの種類自体が異なるため、それぞれに処理を書く必要があると理解しました。
